Préparer son entretien d’embauche

Méthodes, outils et bonnes pratiques pour structurer sa recherche d’emploi, valoriser ses compétences et réussir ses entretiens.

← Retour aux projets

thumbnail.jpg

🧭 Lister ses expériences professionnelles et personnelles

Objectifs :

  • Faire ressortir ses compétences :
    1. Connaissances théoriques
    2. Savoir‑faire
    3. Soft skills
    4. Preuves concrètes
  • Identifier les besoins de l’entreprise
  • Faire apparaître les soft skills
  • Identifier des formations pertinentes

🎯 Plan d’action

  • Se fixer des objectifs
  • Sont‑ils réalisables ?
  • Utiliser les moyens et méthodes :
    • Recherche d’offres d’emploi
    • Suivre une formation
    • Prise de contact
  • Créer un calendrier avec des délais

🔍 Veille d’information

  • Se renseigner sur le secteur et l’entreprise
  • Utiliser LinkedIn
  • Participer à des salons
  • S’abonner à des newsletters

🕵️ Le marché caché de l’emploi (≈ 50 %)

Il passe par l’humain :

  • Job dating
  • Salons
  • Conférences
  • Partenariats d’entreprise
  • Cabinets de recrutement
  • Hackathons

Comme aux Jeux Olympiques : l’important est de participer.

📋 Analyse des offres d’emploi

Les offres contiennent :

  • Profil nécessaire
  • Mission précise
  • Technologies
  • Organisation du travail

Faire un tableau avec :

  • Entreprise
  • Poste
  • Lien
  • Contact
  • Date d’envoi
  • Date de relance
  • Technologies
  • Prestige
  • Domaine
  • Lieu
  • Organisation
  • Contacts internes

Attention

  • Clauses de non‑concurrence
  • Relance : 15 jours après + appel 2 jours plus tard

Exemple d’e‑mail de relance

Bonjour,
Suite à l’envoi de ma candidature le ____ pour le poste de ____ et sans retour de votre part, je me permets de vous recontacter.
Serait-il possible de connaître l’avancement du traitement de cette dernière ?
Je reste à votre disposition pour toute information complémentaire.
En vous remerciant par avance.
Bien cordialement,
Mickael Crozon

📄 CV & lettre de motivation

  • Pour une candidature spontanée : titre = analyste développeur
  • Sinon : reprendre le titre de l’annonce
  • Si l’expérience correspond → la mettre en avant
  • Si les connaissances correspondent → les mettre en premier
  • Noter les compétences de 1 à 5

🎤 Processus de recrutement

  1. CV + lettre
  2. Étude par le chargé de recrutement
  3. Prise de contact
  4. Entretien profil
  5. Entretien technique
  6. Préparation indispensable

Préparation

  • Expliquer des notions complexes
  • Se renseigner sur l’entreprise
  • Préparer un pitch
  • Anticiper les questions
  • Préparer les documents
  • Adapter son style vestimentaire :
    • Start‑up → comme vous êtes
    • Agence web → casual
    • ESN → formel
    • https://www.reddit.com/r/BusinessFashion/

🗣️ Adapter son pitch

Situation Durée Structure Objectif
Rencontre informelle 30 s Qui je suis + Ce que je cherche Créer un contact
Salon / Forum 30–60 s Accroche + Valeur + Action Obtenir un contact
Elevator pitch 30 s – 2 min Problème → Solution → Bénéfice Décrocher un RDV
Job dating 5 min Présentation + Compétences + Motivation Obtenir un entretien
Entretien RH < 3 min Parcours + Réalisations + Projet Montrer sa pertinence
Entretien réseau 2–3 min Contexte + Objectif + Questions Obtenir conseils
Investisseur 5–10 min Problème → Solution → Business Convaincre
Pitch commercial 15–20 min Besoin → Proposition → Preuves Conclure

Phrase de fin :

« Voilà ce que je pouvais vous dire de moi en quelques mots, souhaitez‑vous plus d’informations »

💡 Bonnes pratiques en entretien

  • Ne pas critiquer son ex‑employeur
  • Avoir un discours dynamique
  • Se valoriser sans arrogance
  • Répéter son pitch
  • S’entraîner à l’oral
  • Se chronométrer

Avant / pendant l’entretien

  • Être bien habillé
  • Bloc + stylo
  • Pas de RDV juste après
  • Carte d’identité
  • De quoi payer un café
  • Prévoir le transport
  • Être ponctuel
  • Attitude positive
  • Prendre des notes
  • Poser des questions
  • Répondre de façon structurée
  • Le droit à l’erreur existe
  • Demander la suite du process

🧪 Tests techniques

Entretien développeur

  • Être confiant
  • Admettre quand on ne sait pas
  • Poser des questions
  • S’intéresser aux technologies
  • Demander du feedback

Test d’algorithme (ex : FizzBuzz)

  • Garder les bonnes pratiques
  • Code propre
  • Indentation
  • Nommage
  • Commentaires
  • Pas de copier/coller
  • Expliquer ses décisions

Mini‑projets possibles

  • Intégrer une maquette
  • To‑do list
  • Jeu de morpion
  • API REST

🧮 Notation Big O

  • Recherche dans une liste → log(n)
  • Boucle → n
  • Tri → n log(n)
  • Tri par insertion → n², n³…

🗂️ Structures de données

  • Tableau
  • Pile / File (FIFO)
  • Liste chaînée
  • Table de hachage
  • Arbre (DOM, nœuds)
  • Graphe

🧪 TDD / TCR & qualité du code

  • Commencer un mini‑projet en TDD
  • Ou TCR (Test → Commit → Revert)
  • Pré‑requis : gestion de version
  • Faire du code de qualité
  • Revue de code
  • Coding dojo / kata
  • Respect des standards :
    • PEP (Python)
    • PSR (PHP)
    • KISS
    • SOLID

🎯 Entretien final

  • Poser des questions
  • Montrer qu’on connaît la fourchette salariale
  • Demander du feedback
  • Demander si d’autres contacts recherchent le même profil
  • Rappel : 50 % des offres ne sont pas publiées
