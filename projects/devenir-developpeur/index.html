Le métier de développeur logiciel

🎯 Objectifs du projet

Ce projet vise à :

  • Situer le métier de développeur dans l’entreprise
  • Différencier les principaux types de développeurs
  • Présenter les pistes d’évolution possibles
  • Comprendre les compétences techniques et humaines attendues

Un développeur qui maîtrise les technologies actuelles et possède un profil LinkedIn complet ne manque jamais d’opportunités professionnelles.

🧭 Le quotidien d’un développeur

Même si chaque journée est différente, le cycle de travail reste globalement le même :

  1. Analyser les besoins fonctionnels et techniques
    Réunions, échanges avec le client, recherches.

  2. Imaginer et structurer la solution
    Conception, architecture, validation client.

  3. Implémenter, tester et documenter
    Développement, tests unitaires, documentation claire.

  4. Mettre en production
    Déploiement, vérifications.

  5. Maintenir la solution
    Refactoring, correction de bugs, gestion de la dette technique.

  6. Faire évoluer le produit
    Revues de code, pull requests, pair programming.

Le développeur peut également :

  • Rencontrer les clients pour mieux comprendre leurs besoins
  • Former les utilisateurs à la solution livrée

🌀 La méthode Scrum (Agile)

Scrum est la méthode de gestion de projet la plus utilisée en entreprise.
Elle repose sur 4 valeurs fondamentales :
l’équipe, l’échange, le fonctionnement, la souplesse.

Principes :

  • Le projet est découpé en sprints (1 semaine à 1 mois)
  • Chaque sprint contient des issues (tâches) assignées aux développeurs
  • Tous les matins : daily scrum

    Chaque développeur explique ce qu’il a fait, ce qu’il fera, et ses blocages éventuels.

🧩 Product Owner vs Product Manager

Product Owner (PO)

  • Garant de la valeur immédiate du produit
  • Rédige et priorise les user stories
  • Anime les réunions de planification et de revue

Product Manager (PM)

  • Vision stratégique long terme
  • Analyse du marché et des tendances
  • Définition de la roadmap produit

🏛️ Architecte logiciel

Compétences clés :

  • Design patterns
  • Principes SOLID
  • Architecture hexagonale
  • Microservices vs monolithes
  • Event-driven architecture
  • API REST / GraphQL / gRPC

🧪 Les métiers de la QA

  • Ingénieur QA : conception, exécution, automatisation des tests
  • Analyste QA / Testeur logiciel : plans de test, validation fonctionnelle
  • Responsable QA : stratégie, normes, conformité

  • Savoir se présenter
  • Être honnête sur ses compétences
  • Exemple de test classique : FizzBuzz

    Afficher les nombres de 1 à 100, remplacer les multiples de 3 par “Fizz”, ceux de 5 par “Buzz”, et ceux de 3 et 5 par “FizzBuzz”.

🎓 Se former efficacement

Aujourd’hui, se former est :

  • Simple : tout est accessible en ligne
  • Compliqué : trop de choix, risque de se perdre
  • Jamais magique : sans pratique, rien ne se passe
  • Théorique : les plateformes montrent, mais ne pratiquent pas pour vous

Plateformes utiles :

  • CodinGame
  • CheckiO
  • HackerRank

Il faut un objectif clair pour progresser.
