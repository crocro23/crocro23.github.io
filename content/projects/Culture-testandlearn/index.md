# 🌱 Culture Test & Learn

La culture **test and learn** repose sur 4 piliers essentiels :

1. **L’état d’esprit du développement**  
2. **La sécurité psychologique**  
3. **La gestion de l’échec**  
4. **La data**  

---

# 🧠 1) État d’esprit du développement

Les talents, compétences et qualités **ne sont pas fixes** :

- Nous avons tous la capacité d’apprendre constamment  
- Cela améliore l’engagement des collaborateurs  

### Bonnes pratiques
- Donner des feedbacks réguliers  
- Féliciter l’effort plutôt que le talent  
- Mettre en avant les progrès  
- Développer sa capacité d’écoute  

---

# 🛡️ 2) Sécurité psychologique

Pour apprendre, il faut pouvoir **se tromper dans un cadre bienveillant**.

Questions à se poser :
- Quels feedbacks ai‑je ?  
- Est‑ce facile de demander de l’aide ?  
- Est‑ce facile de parler des problèmes ?  

### Attitudes à adopter
- Soyez ouvert  
- Soyez juste  
- Soyez empathique  
- Soyez exemplaire  

---

# 💥 3) Gestion de l’échec

> *Tout semble parfait… sauf si vous vous êtes trompé.*

- Le **droit à l’erreur** ≠ le **droit à la faute**  
- Le **devoir de tester** insiste sur l’expérimentation avant la décision  

### Objectifs
- Créer un réflexe d’apprentissage  
- Valoriser l’apprentissage  
- Dédramatiser l’échec  

---

# 📊 4) La data

La data permet d’évaluer les idées **objectivement**.

- Chaque collaborateur sait que sa contribution est évaluée sur des données  
- Les décisions basées sur la data objectivent les débats  
- Une culture data peut être développée même dans une petite structure  

### Bonnes pratiques
- Identifier les données nécessaires  
- Questionner les décisions  
- Donner un accès le plus complet possible aux data  

---

# 🧪 Étapes d’un test Test & Learn

## 1) Définir une problématique précise
Elle doit être :
- Définie  
- Précise  
- Intéressante  
- Impactante  

Le KPI doit indiquer **exactement** ce que le test doit mesurer.

Annoncer le test dans l’entreprise permet d’impliquer d’autres services.

---

## 2) Concevoir et déployer le test
- Séparer l’échantillon en **groupe de contrôle** et **groupe de mesure**  

---

## 3) Analyser les résultats
- Nouveau test → nouveau résultat → nouvel apprentissage  
- Le KPI doit mesurer uniquement l’action du test  
- Comparer les moyennes dans les cas simples  
- Déployer la solution étape par étape dans les cas complexes  
- Transformer l’information qualitative en donnée mesurable  
- Réévaluer les hypothèses selon les données  

---

# 📣 Partage des résultats

- Documenter le test selon un format prédéfini  
- Avoir une bibliothèque de cas par thématique et fonction  
- Communiquer sur cette base de test and learn  
- Inclure les résultats dans les formations internes  
- Mettre en valeur **les testeurs (l’équipe, pas l’individu)**  
- Mettre en avant **les apprentissages**, pas seulement les réussites
